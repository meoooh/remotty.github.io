Devise_api

레일스 프로젝트의 인증관련 젬으로 가장 많이 사용하는 것이 devise 젬이다.

또한, 레일스 API 를 작성할 때는 devise의 token_authenticatable 이라는 strategy module을 사용해야 한다.

그러나, Devise 3.1.x 부터는 보안기능이 강화되어 TokenAuthenticatable모듈이 deprecated 되었으며, 더 이상 token_atuthenticatable을 사용할 수 없게 되었다. 이에 대한 제세한 내용은 Devise 3.1: Now with more secure defaults을 참고하기 바란다.

그래서 여기에서는 이러한 문제를 해결하는 방법에 대해서 구글링을 통해 검색한 내용을 근거로 정리해 보도록 하겠다.

위에서 언급했던 블로그의 내용에 의하면 TokenAuthenticatable 모듈에서 제공해 주는 인증 토큰은 digest할 수 없고 어플리케이션마다 필요로하는 보안요구사항이 다양하기 때문에 Devise 3.1.x 부터는 이 모듈를 제거하고 대안으로 2개의 해법을 소개하고 있다.
